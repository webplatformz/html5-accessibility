HTML 5 - Accessibility
05 Aria, HTML 5 & Pitfalls

ARIA - tabindex, autofocus

Ideally: Support tabbing through large pages/sections

Be aware: You can very easily make it a lot worse !

ARIA - role

  • Can be used on any tag
  • Mostly redundant/obsolete with HTML 5
  • Use only where useful / meaningful
  • Screenreaders will probably ignore it

ARIA - role

Good examples

                    
                        <fieldset role="search">...</fieldset>
                        <div role="banner">...</div>
                        <img role="livecam"...>
                        <img role="decorative"...>
                        <div role="glossary">...</div>
                        <div role="alert">...</div>
                        <div role="dialog">...</div>

ARIA - role

Bad examples

                    
                        <p role="text">...</p>
                        <a role="link">...</p>
                        <input type="radio" role="radio">...>
                        <nav role="navigation">...>

HTML 5 - Emphasis

Semantic pitfalls

Never ever nest the following tags ...

  • <article>
  • <nav>
  • <p>
  • Inline Tags (e.g. <a>)
  • <table>
  • <header>
  • <footer>
  • <main>
  • <aside>
At most one <nav> per page

Semantic pitfalls

Never ever do meaningless combinations/nestings like ...

  • <p><article>...
  • <p><li>...
  • <p><ul|ol><li>...
  • <li> without enclosing <ul|ol>
  • <dl> without enclosing <dd>

Semantic pitfalls

  • Always provide alt= for images
  • Avoid href="javascript:..."
  • Avoid empty href="" or href="#"
  • Avoid empty tags <p> </p>
  • Contrast-ful styles for :focus and :hover
  • Ignored: display:none and visibility:hidden
