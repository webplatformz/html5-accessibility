Screenreaders in 20xx

  • Limited to text only: fixed
  • Not aware of interaction: fixed
  • No support for Javascript: fixed
  • Not aware of animations: fixed
Getting better & smarter every year

(General) Behaviour of Screenreaders

  • Recall: There is no standard !
  • Every Screenreader behaves slightly different
  • They might/will announce certain tags and attributes
Do not abuse tags for style & layout !

Announcing tags

  • <nav> "main navigation"
  • <aside> "side-content" / "complementary content"
  • <h1> "page title" / "main title"
  • <h2-6> "sub title"
  • <article> "article"
At most one  <nav> and <h1>

Announcing tags

  • <a> "link" / "clickable"
  • <img> "image"
  • <a><img> "link graphic"
  • <bold> "bold"
  • <em> "emphasized"
  • ...

Announcing attributes

They might/will announce certain attributes

  • <a NO_ALT> "link to ${href}" :(
  • <a><img NO_ALT> "link to ${href}" :(
  • role="search" "start of search" / "end of search"
  • type="tel" "input of type telephone number"

Announcing pitfalls

  • If no alt is provided they might read the href
  • Omit redundant prefix "link to" or "image of" in alt
  • <table> and tabular-like <div>-structures
  • Never ever nest <article>
  • Never ever nest <nav>
Good accessibility depends on meaningful structure & content !

(General) Best Practice

  • Start small
  • Keep it simple
  • Be minimalistic
  • Go for meaningful semantic HTML
  • Provide "Fallbacks"

Personal Preference

  • Strict (X)HTML 5
  • Strict Javascript only
  • Strict Css only

Pro & Cons of Strictness

  • No empty tags
  • No empty attributes
  • No non-closing tags
  • No invalid html, css, js
  • Strict quoting
  • Strict encoding
  • More effort
  • Incompatibility
Maximum validity, predictability, security

Semantic HTML != Meaningful

Semantic & meanigful HTML

  • DOM Structure
  • No excessive/useless tags & attributes
  • No omission of relevant tags & attributes
Screenreaders evaluate (mostly only) the structure of tags & attributes

Document

You MUST provide a valid context for the document

  • <!doctype html>
  • <html lang="en">
  • <meta charset="utf-8">
Changing this after initial load might not work properly

noscript

Forget about <noscript> - that is about javascript not accessibility !

Modern Screenreaders should be able to deal with Javascript

noscript ... BUT

A reader-friendly, clutter-free view is mostly appreciated by people with disabilities ;)
